# Hands-on Labs – Day 01

## Azure Landing Zone Implementation

### Estimated Duration: 4 Hours

## Overview

In this lab, you will explore the Azure Landing Zone (ALZ) framework, its governance components including Management Groups, Policies, and Subscriptions, and the benefits of automation. You will deploy ALZ using ARM templates, review the deployed foundation, add subscriptions to management groups, deploy the App Service Landing Zone Accelerator, and validate network configurations. By the end, you will have hands-on experience in setting up a secure, scalable Azure environment for enterprise workloads.

## Objective

In **Day-1** of this workshop you will learn how to:

- Understand Azure Landing Zone concepts and governance components.
- Deploy Azure Landing Zone using ARM templates and Azure DevOps.
- Review deployed Management Group hierarchy, subscriptions, and policy compliance.
- Add subscriptions to App Landing Zone Management Groups and assign policies.
- Deploy Application Workloads using App Service Landing Zone Accelerator.
- Validate network configurations including VNet peering, DNS zones, and security groups.

## Prerequisites

Participants should have:

- Basic understanding of Azure governance and resource management.
- Familiarity with Azure Portal, ARM templates, and Azure DevOps.
- Experience with Azure subscriptions, management groups, and policies.
- Awareness of networking concepts like VNet, peering, and DNS in Azure.
- Access to the lab-provided Azure tenant and administrative permissions.

## Explanation of Components

- **Azure Landing Zone (ALZ)**: A framework for deploying scalable, secure Azure environments with built-in governance.
- **Management Groups**: Hierarchical containers for organizing subscriptions and applying policies at scale.
- **Azure Policy**: Service for enforcing organizational standards and assessing compliance.
- **ARM Templates**: Declarative files for deploying Azure resources consistently.
- **App Service Landing Zone Accelerator**: Pre-built solution for deploying secure App Service environments with networking and security features.
- **VNet Peering and DNS**: Networking configurations for secure connectivity between Azure resources.
